---

## 🎯 당신의 역할: "비전 제시자 (The Visionary)"

당신은 **작곡가**입니다. 아름다운 곡(로드맵)을 작곡하면, Claude Code가 **연주자**로서 완벽하게 연주(구현)합니다.

### ✅ 당신이 할 일 (20% - 핵심 가치)

1. **📋 비전 및 로드맵 작성**
   - 어디로 가야 하는가? (방향)
   - 무엇을 만들어야 하는가? (목표)
   - 왜 필요한가? (가치)

2. **📋 사용자 시나리오 작성**
   - 누가 사용하는가?
   - 어떤 문제를 해결하는가?
   - 어떻게 사용하는가?

3. **📋 품질 기준 제시**
   - 성공의 기준은?
   - 어떤 메트릭으로 측정?
   - 위험 요소는?

4. **📋 완성 후 리뷰**
   - 요구사항 충족 여부
   - 개선 제안
   - 다음 단계 방향

### ❌ 당신이 하지 않아도 되는 일 (80% - Claude Code가 처리)

1. ❌ 직접 코드 작성
2. ❌ 파일 수동 편집
3. ❌ Git 커맨드 실행
4. ❌ 빌드 검증
5. ❌ import 경로 수정
6. ❌ 타입 오류 해결

**왜?** Claude Code가 15배 빠르고, 오류율이 1/4이기 때문입니다.

## 🔄 협업 워크플로우

### Step-by-Step 가이드

#### Step 1: 당신이 비전을 제시합니다
```
📋 docs/FEATURE_PHASE_2_AUTOMATION.md 작성
  ├─ 목표: Time Tracking 자동 연동
  ├─ 시나리오: 작업 지연 → 불확실성 증가
  ├─ 요구사항: 3가지
  └─ 성공 기준: Toast 알림 표시
```

**소요 시간**: 30분 - 1시간

#### Step 2: Claude Code가 질문합니다
```
💬 "설계를 검토했습니다. 3가지 질문이 있습니다:"
  ├─ Q1: Time Tracking 주기는 얼마나 자주?
  ├─ Q2: 임계값은 몇 %?
  └─ Q3: UI 알림 스타일은?
```

**응답 방법**: 같은 파일을 업데이트하세요

#### Step 3: 당신이 답변합니다
```
📋 FEATURE_PHASE_2_AUTOMATION.md 업데이트
  ├─ A1: 1분마다 체크
  ├─ A2: 예상 시간의 150%
  └─ A3: Toast 알림 + 빨간색 Badge
```

**소요 시간**: 10분

#### Step 4: Claude Code가 구현합니다
```
💻 자동으로:
  ├─ 1. 파일 탐색 및 생성
  ├─ 2. 코드 작성
  ├─ 3. 테스트 실행
  ├─ 4. 빌드 검증
  ├─ 5. Git 커밋
  └─ 6. Obsidian 동기화
```

**소요 시간**: 1-3시간 (자동)
**당신의 개입**: 0분

#### Step 5: 당신이 확인합니다
```
브라우저에서:
  ├─ http://localhost:3000 접속
  ├─ 시나리오대로 동작하는지 확인
  └─ 만족하면 ✅, 개선 필요하면 📋 리뷰 작성
```

**소요 시간**: 10-20분

#### Step 6: (필요시) 당신이 리뷰합니다
```
📋 docs/REVIEW_2025-11-27.md 작성
  ├─ ✅ 요구사항 충족
  ├─ ✅ 시나리오 동작
  └─ 💡 개선 제안: Toast 알림 색상 변경
```

**소요 시간**: 15분

#### Step 7: Claude Code가 개선합니다
```

  └─ ✅ 완료
```

**소요 시간**: 5-10분 (자동)
