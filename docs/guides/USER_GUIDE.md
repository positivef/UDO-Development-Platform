## 🎛️ 컴포넌트별 사용법

### 1. System Status (좌측 상단)

**기능**: 시스템 리소스 모니터링

**표시 정보**:
- CPU 사용률 (%)
- 메모리 사용량 (%)
- UDO 시스템 상태
- AI 서비스 상태

**사용법**:
- 자동 업데이트 (30초마다)
- 별도 조작 불필요

**활용**:
- 시스템 과부하 감지
- 리소스 최적화 시기 판단

---

### 2. Phase Progress (좌측 하단)

**기능**: 개발 단계 관리

**5단계 프로세스**:
1. 💡 **Ideation** - 아이디어 구상 및 요구사항 정의
2. 📐 **Design** - 시스템 설계 및 아키텍처
3. 🚀 **MVP** - 최소 기능 제품 개발
4. 💻 **Implementation** - 전체 기능 구현
5. 🧪 **Testing** - 테스트 및 검증

**사용법**:
1. 현재 단계 확인 (강조 표시됨)
2. 단계 클릭으로 전환 가능
3. 각 단계별 진행률 확인

**활용**:
- 프로젝트 진행 상황 추적
- 단계별 작업 계획
- 팀원과 진행 상황 공유

---

### 3. Uncertainty Map (중앙 상단)

**기능**: 불확실성 시각화 및 신뢰도 표시

**불확실성 상태** (4단계):
- 🟢 **Deterministic** (확정적): 90%+ 신뢰도
  - 명확한 요구사항
  - 검증된 기술 스택
  - 예측 가능한 결과

- 🟡 **Probabilistic** (확률적): 70-90% 신뢰도
  - 일부 불확실성 존재
  - 여러 해결 방안 가능
  - 리스크 관리 필요

- 🟠 **Chaotic** (혼돈): 50-70% 신뢰도
  - 높은 복잡도
  - 예측 어려움
  - 실험적 접근 필요

- 🔴 **Void** (미지): <50% 신뢰도
  - 전혀 알 수 없음
  - 연구 개발 필요
  - 단계적 접근 필수

**사용법**:
- 현재 상태 확인
- 신뢰도 퍼센트 모니터링
- 권장 액션 참고

**활용**:
- 리스크 평가
- 개발 전략 수립
- 의사결정 지원

---

### 4. AI Collaboration (우측 상단)

**기능**: 3-AI 협업 상태 모니터링

**AI 서비스**:
1. 🤖 **Claude** (Anthropic)
   - 역할: 계획 및 설계
   - 강점: 체계적 사고, 안전성

2. 🧠 **Gemini** (Google)
   - 역할: 실행 및 검증
   - 강점: 빠른 처리, 다양한 도구

3. 💬 **ChatGPT** (OpenAI)
   - 역할: 리뷰 및 개선
   - 강점: 창의성, 문제 해결

**표시 정보**:
- 각 AI 상태 (Active/Idle)
- 최근 작업
- 응답 시간
- 사용 통계

**사용법**:
- 실시간 상태 확인
- 자동 업데이트 (30초마다)

**활용**:
- AI 서비스 가용성 확인
- 성능 모니터링
- 협업 효율성 추적

---

### 5. Control Panel (우측 하단)

**기능**: 작업 실행 및 제어

**사용법**:

#### Step 1: 작업 입력
```
┌─────────────────────────────────────┐
│ Task: "데이터베이스 스키마 생성"     │
└─────────────────────────────────────┘
```

#### Step 2: 단계 선택
- 현재 단계에서 실행 (기본)
- 또는 다른 단계 선택

#### Step 3: 우선순위 설정
- Normal (기본)
- High (긴급)
- Low (여유)

#### Step 4: Execute 버튼 클릭
- WebSocket으로 실시간 실행
- 결과는 Execution History에 표시

**예시**:
```
Task: "Quality Metrics 대시보드 생성"
Phase: Implementation
Priority: Normal
→ [Execute] 클릭
→ 실행 결과 확인
```

**활용**:
- 빠른 작업 실행
- 자동화된 개발 워크플로우
- 실시간 피드백

---

### 6. Execution History (하단)

**기능**: 실행된 작업 히스토리

**표시 정보**:
- 작업 내용
- 실행 시간
- 성공/실패 상태
- 성능 메트릭

**사용법**:
1. 최근 작업 확인
2. 성공/실패 필터링
3. 상세 정보 확장
4. 타임라인 추적

**활용**:
- 작업 이력 관리
- 문제 디버깅
- 성능 분석
- 작업 패턴 파악

---

### 7. Version History (최하단)

**기능**: Git 커밋 히스토리 ✅

**표시 정보**:
- 최근 20개 커밋
- 커밋 메시지
- 작성자 및 이메일
- 날짜 및 시간
- 파일 변경 통계
  - 📝 Modified files
  - ➕ Lines added (녹색)
  - ➖ Lines deleted (빨강)
- 브랜치 및 태그

**커밋 타입별 색상**:
- 🟢 `feat:` - 새 기능
- 🔴 `fix:` - 버그 수정
- 🔵 `refactor:` - 리팩토링
- 🟣 `docs:` - 문서
- 🟡 `test:` - 테스트

**사용법**:
1. 커밋 클릭으로 확장/축소
2. 파일 변경 상세 보기
3. 브랜치 확인
4. 자동 업데이트 (60초마다)

**활용**:
- 코드 변경 추적
- 팀원 작업 확인
- 릴리즈 노트 작성
- 버그 원인 추적

## 💻 Quality Metrics 페이지 (/quality)

### 접근 방법

1. **메인 대시보드에서**:
   - 우측 상단 "Quality Metrics" 버튼 클릭

2. **직접 접속**:
   - http://localhost:3000/quality

### 화면 구성

```
[⬅️ Back to Dashboard]

┌─────────────────────────────────────────────┐
│ 💻 Code Quality Dashboard                   │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│ Overall Score: 8.5/10  [🔄 Refresh]         │
│ ████████████████████░░░░                    │
└─────────────────────────────────────────────┘

┌──────────────────────┬──────────────────────┐
│ 🐍 Python            │ 📘 TypeScript        │
│ Score: 9.2/10        │ Score: 8.8/10        │
│ Issues: 12           │ Files: 45            │
└──────────────────────┴──────────────────────┘

┌─────────────────────────────────────────────┐
│ 🧪 Test Coverage                            │
│ 85.5%  ████████████████░░░                  │
│ Passed: 118/120  Failed: 2                  │
└─────────────────────────────────────────────┘
```

### 사용법

#### 1. 전체 품질 점수
- **Overall Score**: Python(30%) + TypeScript(30%) + Coverage(40%)
- 색상 코딩:
  - 🟢 9.0-10.0: Excellent
  - 🔵 7.0-8.9: Good
  - 🟡 5.0-6.9: Fair
  - 🔴 0.0-4.9: Poor

#### 2. Python 코드 품질 (Pylint)
- **점수**: 0-10
- **이슈 타입**:
  - Convention (컨벤션 위반)
  - Refactor (리팩토링 필요)
  - Warning (경고)
  - Error (에러)
  - Fatal (치명적)

**설치 필요**:
```bash
cd backend
pip install pylint
```

#### 3. TypeScript 코드 품질 (ESLint)
- **점수**: 0-10
- **통계**:
  - 분석된 파일 수
  - 에러 개수 (빨강)
  - 경고 개수 (노랑)

**이미 설치됨** (web-dashboard)

#### 4. 테스트 커버리지 (pytest-cov)
- **커버리지**: 0-100%
- **테스트 결과**:
  - Total tests
  - Passed (녹색)
  - Failed (빨강)
  - Success rate

**설치 필요**:
```bash
cd backend
pip install pytest pytest-cov
```

#### 5. Refresh 버튼
- 품질 분석 재실행
- 5-30초 소요
- 실시간 업데이트

#### 6. Quick Actions
- Run Pylint (향후 구현)
- Run ESLint (향후 구현)
- Run Tests (향후 구현)
- View Report (향후 구현)

## 🛠️ 문제 해결

### Q1: 대시보드가 로딩만 계속됨

**원인**: 백엔드 서버 미실행

**해결**:
```bash
cd backend
python main.py
```
확인: http://localhost:8000/docs

### Q2: Version History가 비어있음

**원인**: Git 저장소가 아니거나 커밋 없음

**해결**:
```bash
git init
git add .
git commit -m "Initial commit"
```

### Q3: Quality Metrics에서 에러 표시

**원인**: Pylint/pytest 미설치

**해결**:
```bash
cd backend
pip install pylint pytest pytest-cov
```

### Q4: WebSocket 연결 안 됨 (🔴 Disconnected)

**원인**: 백엔드 서버 WebSocket 미지원

**해결**:
- 백엔드 서버 재시작
- 브라우저 새로고침

### Q5: 화면이 너무 자주 새로고침됨

**현재 설정**: 30초/60초로 조정됨

**더 느리게**:
- `dashboard.tsx` 파일에서
- `refetchInterval` 값 증가 (예: 60000, 120000)
