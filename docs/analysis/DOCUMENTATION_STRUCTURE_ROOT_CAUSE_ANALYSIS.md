## Executive Summary

**Problem**: Severe documentation organization crisis with 100+ files scattered across `docs/` and minimal `claudedocs/` usage, violating RULES.md:168 principle.

**Root Cause**: **Lack of enforcement mechanism** for RULES.md documentation placement rules + **unclear definition** of "Claude-specific" vs "permanent project documentation".

**Impact**:
- Knowledge fragmentation across 26+ subdirectories
- Duplicate/overlapping content (progress reports vs completion summaries vs analysis)
- AI context confusion (where to write reports?)
- Developer onboarding friction (which docs to trust?)

**Recommendation**: 3-tier restructuring with automated enforcement

## 2. Root Cause Analysis (5 Whys)

### Why are docs misorganized?
**Because** 100+ files are in `docs/` instead of `claudedocs/`

### Why are files in wrong location?
**Because** no automated enforcement of RULES.md:168 placement rule

### Why no enforcement?
**Because** RULES.md is a guideline document, not executable pre-commit hook

### Why not executable?
**Because** unclear definition of "Claude-specific" vs "permanent" documentation

### Why unclear definition?
**Because** project documentation taxonomy was never formally defined

**Root Cause**: **Missing documentation governance framework** with:
1. Clear taxonomy (what goes where)
2. Automated enforcement (pre-commit hooks)
3. Migration strategy (existing â†’ organized)

## 3. Document Taxonomy Definition

### 3.1 `claudedocs/` Criteria (RULES.md:168 Enforcement)

**Move to `claudedocs/` if ANY of these apply**:

1. **Temporal Markers**:
   - File name contains dates: `*_2025-12-13.md`, `WEEK0_*.md`
   - Content is session-specific: "Completion Summary", "Progress Report"
   - Relevance window: <4 weeks

2. **AI-Generated Reports**:
   - Created BY AI (Claude) FOR AI (future sessions)
   - Not intended for human developers as primary documentation
   - Examples: worklogs, analysis reports, completion summaries

3. **Session Ephemeral**:
   - Handoff notes, recovery logs, debug traces
   - Pre-commit validation reports
   - One-time architectural analysis (not design decisions)

4. **Explicit AI Attribution**:
   - File name contains: `CLAUDE_`, `_WORKLOG`, `_HANDOFF`
   - Content header: "Generated by Claude", "AI-generated"

### 3.2 `docs/` Criteria (Permanent Project Documentation)

**Keep in `docs/` if ALL of these apply**:

1. **Permanent Value**:
   - No date stamps in filename
   - Reusable templates, guides, strategies
   - Reference documentation (not reports)

2. **Human-Authored or Reviewed**:
   - Design decisions (reviewed by humans)
   - User guides (for end users)
   - API specifications (machine-readable contracts)

3. **Version-Controlled Knowledge**:
   - Architectural decisions (ADRs)
   - PRD documents (product requirements)
   - Integration strategies (foundational)

4. **Developer Onboarding**:
   - Required reading for new team members
   - Feature documentation (permanent reference)

### 3.3 Gray Area Resolution Rules

| Scenario | Location | Rationale |
|----------|----------|-----------|
| `*_IMPLEMENTATION_SUMMARY.md` (dated) | `claudedocs/` | Temporal, AI-generated |
| `*_IMPLEMENTATION_GUIDE.md` (no date) | `docs/guides/` | Permanent, reusable |
| `PREMORTEM_ANALYSIS_2025-12-06.md` | `claudedocs/analysis/` | Dated, one-time analysis |
| `ARCHITECTURE_EXECUTIVE_SUMMARY.md` | `docs/architecture/` | Permanent, design decision |
| `WEEK0_COMPLETION_SUMMARY.md` | `claudedocs/progress/` | Temporal, session-specific |
| `KANBAN_INTEGRATION_STRATEGY.md` | `docs/features/kanban/` | Permanent, foundational |

## 6. Root Cause Attribution

### Primary Root Causes (in priority order):

1. **Absence of Automated Enforcement** (40% contribution)
   - No pre-commit hook to check file placement
   - No CI/CD validation of documentation structure
   - RULES.md is advisory, not enforced
   - **Impact**: Files land in wrong locations by default

2. **Unclear Taxonomy** (30% contribution)
   - "Claude-specific" definition too vague
   - No decision tree for placement
   - Gray areas (Implementation Summary, Analysis Reports)
   - **Impact**: AI and humans guess placement incorrectly

   - 33+ files in `docs/sessions/progress/` from early development
   - No migration strategy when RULES.md was added
   - Files created before `claudedocs/` convention existed
   - **Impact**: Old files never migrated

   - Claude Code defaults to `docs/` (visible in file trees)
   - `claudedocs/` hidden or less prominent
   - No template files in `claudedocs/` to guide placement
   - **Impact**: Easier to create in wrong location
