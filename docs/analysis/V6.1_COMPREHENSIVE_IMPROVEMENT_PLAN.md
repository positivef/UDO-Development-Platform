## 3. Uncertainty Map 리스크 평가

### 3.1 전체 불확실성: 🔴 CHAOTIC (62%)

```python
UncertaintyVector(
    technical=0.47,   # 🟠 QUANTUM
    market=0.28,      # 🟢 PROBABILISTIC
    resource=0.88,    # 🔴 CHAOTIC
    timeline=0.91,    # ⚫ VOID
    quality=0.53      # 🟠 QUANTUM
)
→ magnitude = 62% (CHAOTIC)
→ dominant_risk = timeline (91% VOID)
```

**해석**: "Butterfly effect - 작은 변화가 큰 편차 유발"

**예측**:
- Week 0 없이: **80% 실패 확률** (Week 6)
- Week 0 적용: **70-85% 성공 확률** (Week 14)

## 6. 측정 가능한 KPI (주관적 → 객관적)

### 6.1 문제: 주관적 성공 기준

**현재 V6.1** (40%가 주관적):
- "뭔가 동작한다" ← 무엇이 "동작"인가?
- "믿을 만하다" ← 누가 믿는가? 얼마나?
- "유용하다" ← 어떻게 측정?
- "지식이 자산이다" ← 자산의 정의는?

**문제점**:
- 검증 불가능
- PM/개발자마다 다른 해석
- 잘못된 확신 → 취약한 기반 위에 구축

### 6.2 해결책: 객관적 메트릭

| 주관적 기준 | 객관적 메트릭 | 측정 방법 |
|-------------|---------------|-----------|
| "뭔가 동작한다" | 핵심 기능 100% smoke test 통과 | pytest smoke_tests/ |
| "믿을 만하다" | 개발자 신뢰도 ≥75% (설문) | 5점 척도 설문 (n≥10) |
| "유용하다" | 일일 활성 사용자 ≥3명 | 로그 분석 |
| "지식이 자산이다" | 지식 재사용율 ≥90% | (tier1_hits/total) × 100% |
| "예측 정확도 40%" | correct/total ≥ 0.40 | Ground truth 비교 |
| "자동화율 95%" | AI assistance rate 측정 | 1주일 시간 추적 |

**구현**: Week 0 Day 1 (0.25일)
