# AI 개발 명령 프롬프트 가이드

## 🎯 UDO 기반 개발 시 AI에게 명령하는 방법

---

## 📋 프롬프트 기본 구조

```markdown
[1. 컨텍스트 제공]
[2. 구체적 작업 요청]
[3. 규칙/제약 조건]
[4. 예상 산출물]
```

---

## 🚀 상황별 프롬프트 템플릿

### 1. 새 기능 개발 요청

```markdown
# 컨텍스트
이 프로젝트는 UDO Development Platform 기반입니다.
- 현재 Tier: [1/2/3/4]
- CLAUDE.md와 AGENTS.md 규칙을 따르세요.
- .governance.yaml 설정을 준수하세요.

# 작업 요청
[기능명]을 구현해주세요.

# 요구사항
- [ ] 요구사항 1
- [ ] 요구사항 2
- [ ] 테스트 코드 포함

# 제약 조건 (UDO 규칙)
- 파일 3개 이상 수정 시 설계 문서 필요 (P1)
- Black 포맷팅 적용
- pytest 테스트 포함
- 한글 주석 허용

# 예상 산출물
- src/[기능명].py
- tests/test_[기능명].py
- docs/[기능명]_DESIGN.md (3+ 파일 시)
```

---

### 2. 버그 수정 요청

```markdown
# 버그 설명
[버그 현상 설명]

# 재현 방법
1. [단계 1]
2. [단계 2]

# 관련 파일
- [파일 경로]

# 요청
이 버그를 수정하고 재발 방지 테스트를 추가해주세요.

# UDO 규칙 준수
- 커밋 메시지: "fix: [버그 설명]"
- 테스트 추가 필수
```

---

### 3. 리팩토링 요청

```markdown
# 리팩토링 대상
[파일/함수 경로]

# 문제점
- [현재 문제 1]
- [현재 문제 2]

# 개선 방향
- [개선 방향 1]
- [개선 방향 2]

# 제약 조건
- 기존 테스트 통과 유지
- API 호환성 유지
- 설계 문서 업데이트 (P1 규칙)
```

---

### 4. 세션 시작 시 컨텍스트 공유

```markdown
# 프로젝트 현황
- 프로젝트: [프로젝트명]
- Tier: [현재 Tier]
- 마지막 작업: [이전 세션 작업 내용]

# 참조 문서
- CLAUDE.md 읽어주세요
- claudedocs/HANDOFF.md 확인해주세요

# 오늘 목표
1. [목표 1]
2. [목표 2]

# 주의사항
- .governance.yaml 규칙 준수
- 예정된 작업 확인: python scripts/session_start.py
```

## 💡 효과적인 프롬프트 작성 팁

### ✅ Do's

```markdown
# 좋은 예시 1: 구체적 + 규칙 명시
"src/auth/login.py에 2FA 기능을 추가해주세요.
 - CLAUDE.md의 코딩 스타일 따르기
 - pytest 테스트 포함
 - 커밋 메시지: feat: 2FA 로그인 추가"

# 좋은 예시 2: 컨텍스트 + 제약조건
"이 프로젝트는 UDO Tier 2입니다.
 Kanban API에 필터링 기능을 추가해주세요.
 - 파일 2개 이내로 수정
 - 기존 테스트 통과 확인
 - Black 포맷팅 적용"
```

### ❌ Don'ts

```markdown
# 나쁜 예시 1: 너무 모호
"로그인 기능 만들어줘"
→ 개선: "FastAPI 기반 JWT 로그인 API를 만들어주세요.
         backend/app/routers/auth.py에 추가."

# 나쁜 예시 2: 규칙 무시
"빨리 기능 추가해줘, 테스트는 나중에"
→ 개선: "기능 추가하고 기본 테스트도 포함해주세요.
         Tier 1이라 커버리지 60% 목표입니다."
```

## 📝 복사해서 쓰는 표준 프롬프트

### 프롬프트 A: 새 기능

```
이 프로젝트는 UDO Development Platform (Tier 2) 기반입니다.

[기능 설명]을 구현해주세요.

요구사항:
1. [요구사항]
2. 테스트 코드 포함
3. CLAUDE.md 스타일 준수

산출물:
- 소스 코드
- 테스트 코드
- (3+ 파일 시) 설계 문서
```

### 프롬프트 B: 버그 수정

```
버그 수정 요청입니다.

문제: [버그 현상]
파일: [관련 파일]

수정하고 재발 방지 테스트 추가해주세요.
커밋 메시지: "fix: [설명]"
```

### 프롬프트 C: 코드 리뷰

```
다음 코드를 UDO 규칙 기준으로 리뷰해주세요:

[코드]

체크 포인트:
- Black 포맷팅
- Flake8 규칙
- 테스트 존재 여부
- 한글 주석 적절성
```

### 프롬프트 D: 설계 문서 요청

```
[기능명]에 대한 설계 문서를 작성해주세요.

UDO P1 규칙 (Design Review First)에 따라:
- 변경 영향 분석
- 위험 요소 식별
- 테스트 전략

docs/[기능명]_DESIGN_REVIEW.md 형식으로 작성.
```
